# 商標、名稱與非背書政策

## 1. 範圍

「Stock Danmaku」、「彈幕 Overlay」、專案標誌、官方圖示、官方網域、套件外觀及相關識別（以下合稱「專案識別」）用來辨認原專案。本文不表示任何標誌已在特定司法管轄區註冊。

[LICENSE](LICENSE) 授予的是有限的著作權／專利權利，**不授予商標、商號、服務標章、網域或商品外觀權利**。

## 2. 通常允許的真實描述

在不造成混淆、不暗示背書的前提下，可以：

- 說明「此專案是 Stock Danmaku 的 fork」或「基於 Stock Danmaku」；
- 連結原始 source、LICENSE 或比較相容性；
- 在評論、研究、新聞、教學或相容性文件中以文字提及專案名稱；
- 保留 LICENSE、著作權與來源聲明所要求的歸屬資訊。

不得把歸屬義務誤解為使用官方品牌發布修改版的權利。

## 3. 修改版與自架服務必須做的事

若散布修改版或向第三人提供修改版功能：

1. 使用足以避免一般使用者混淆的主要名稱、圖示、套件描述與視覺識別。
2. 在 README、關於頁、商店 listing、安裝畫面與服務入口醒目標示「非官方修改版」。
3. 說明修改者／營運者是誰，提供修改紀錄與相應 source URL。
4. 使用自己的網域、發布帳號、簽章憑證及支援管道。
5. 發布到 Android／桌面／瀏覽器平台時，使用自己的 application identifier、bundle identifier、extension ID 或其他唯一識別；不得讓更新通道與官方版本混淆。

可使用的描述範例：

> Community Overlay — 非官方版本，基於 Stock Danmaku；未獲原專案背書。

## 4. 禁止使用

不得：

- 把第三方版本、服務、商店頁、repository、網域或社群帳號稱為「官方」；
- 明示或暗示原作者、貢獻者或維護者認可、贊助、保證、稽核或支援；
- 使用近似名稱、圖示、網域或版面，使一般使用者誤認來源；
- 移除修改標示、偽造簽章、冒用官方發布者名稱或安全聲明；
- 用專案識別從事 LICENSE 禁止的商用活動。

## 5. 原樣鏡像

即使只是原樣鏡像，也必須清楚標示鏡像營運者、來源 commit／版本與「非官方鏡像」，不得暗示鏡像由原專案控制。不得用修改過的 binary 冒充原樣鏡像。

## 6. 權限與詢問

本政策未明確允許的品牌使用，需要相關權利人的事前書面同意。請只使用 repository profile 或官方網站實際列出的聯絡管道；本文件不提供虛構地址。

本政策不是法律意見。名稱或標誌是否構成合理指稱、混淆或侵權，取決於事實與適用法律；正式發布前請由合格律師審閱。
