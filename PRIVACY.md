# 隱私政策

版本：1.0｜生效日期：2026-07-15

本文件說明本專案官方發行版本的設計，以及由專案維護者明確標示為「官方」的服務如何處理資料。第三方 fork、自架服務、瀏覽器商店重包裝或 reverse proxy 由各自營運者控制；其資料處理不受本政策保證，也不代表官方。

## 1. 核心原則

- 不建立使用者帳號。
- 極簡網站不初始化聊天室 Socket，不使用第三方分析、廣告、遠端字型或遠端執行程式碼。
- 擴充功能不讀取、上傳或分析宿主頁面正文、表單、輸入框、cookie、localStorage 或網址參數。
- 不出售個人資料，不用聊天內容做廣告定向。
- 僅處理提供房間、彈幕、檢舉、防濫用與安全維運所需資料。

## 2. 會處理的資料

### 2.1 連線與房間

使用客戶端會送出：

- 隨安裝產生的穩定 `clientId`（假名識別，不是帳號）；
- 平台類型、目前房間、8 位房間碼；
- 暱稱、彈幕文字、顏色及必要時間戳；
- 建立／管理房間所需的名稱、公開狀態、保留選項與密碼操作；
- Socket 連線所必然帶有的 IP 位址及協定 metadata。

伺服器在記憶體中使用 IP 與 clientId 執行連線容量、速率限制、重複內容及濫用防護。房間資料庫會保存房間 metadata，以及建立者 IP、clientId 與 owner credential 的單向雜湊。房間密碼以加鹽 scrypt 衍生值保存，不保存明文。

雜湊值仍可能具可連結性，不能視為完全匿名資料。

### 2.2 訊息與最近歷史

每個活動房間最多保留最近 200 則已送達訊息於伺服器記憶體，供即時顯示與最近歷史使用。一般訊息歷史不寫入持久化訊息資料表；服務重啟、房間生命週期結束或記憶體狀態釋放時會消失。

不要在彈幕中輸入姓名、聯絡方式、憑證、財務資料、健康資料或其他敏感資訊。房間不是端對端加密私人空間；同房成員及服務營運者可能看到內容。

### 2.3 檢舉

檢舉時，伺服器會持久化：

- 被檢舉訊息文字、message ID、房間碼與檢舉理由；
- 檢舉者及目標的假名化 session/client 識別；
- 檢舉者 IP 的單向雜湊；
- 建立時間與處理狀態。

檢舉資料用於去重、濫用處理與安全稽核。現行應用程式不自動為檢舉紀錄設定到期日；紀錄保留至營運者完成處理並刪除、輪替資料庫或終止服務。官方營運者應定期最小化與清理；自架者必須自行制定保留政策。

### 2.4 本機資料

- **Android：** 一般設定、clientId、暱稱、房間與位置存在 app private preferences；owner credential 經 Android Keystore 金鑰加密後保存。
- **Windows／macOS：** 一般設定與房間狀態存在應用程式本機儲存；owner credential 存在作業系統 credential vault/keyring。
- **Chrome／Edge：** 一般設定、clientId、房間與 UI 狀態存在 extension storage；owner credential 僅應由 background／可信 extension context 存取，content script 與宿主頁面不應取得其值。

清除應用程式或擴充功能資料會刪除本機狀態，並可能永久失去無帳號房主權限。請勿把 owner credential 放入截圖、console、Issue 或同步筆記。

## 3. 擴充功能網站權限

擴充功能要求一般 HTTP／HTTPS 網站範圍，目的是自動建立 Overlay。content script 以隔離執行環境與 Shadow DOM 顯示 UI；background service worker 維持共用 Socket。專案不建立 `window.postMessage` bridge，不讀取頁面內容，也不把頁面資料送到後端。

瀏覽器權限本身的技術能力大於上述用途；安裝前請閱讀 [docs/extension-permissions.md](docs/extension-permissions.md)，並只安裝可核對來源的版本。

## 4. 極簡網站與基礎設施

專案應用程式不內建 analytics、廣告或追蹤像素。HTTP/TLS 提供者、CDN、reverse proxy、作業系統或託管商仍可能產生標準連線／安全日誌；官方營運與自架營運者應限制欄位、權限與保留時間，不得記錄 owner credential、房間密碼、Socket auth 或完整錯誤 payload。

`/healthz` 只回傳最小可用狀態，不應揭露 PID、版本、資料庫、記憶體、房間數或部署細節。

## 5. 資料用途與分享

資料僅用於：提供即時房間功能、保存必要房間 metadata、執行 typed ACK／排隊、檢舉與防濫用、偵錯安全事件，以及遵守具拘束力的法律要求。

官方不出售資料。只有在服務供應商為提供基礎設施所必要、調查安全／濫用事件、取得使用者指示，或依法必須時才會有限分享。第三方房間參與者能看到該房間傳送的內容。

## 6. 控制與請求

你可以：

- 停止 Overlay、離開房間、撤銷 extension 網站權限；
- 清除 app／extension 本機資料；
- 對官方營運者提出存取、更正或刪除可合理定位的持久資料請求。

因本服務沒有帳號，營運者可能無法僅憑暱稱確認資料屬於你，也不會要求你公開 owner credential。提出隱私請求時，請使用官方網站或 repository profile 實際列出的私下聯絡方式，不要把敏感資訊貼在公開 Issue。

法令、資安事件調查、權利主張或防止濫用所必須的紀錄，可能依法保留。

## 7. 兒童與敏感內容

本服務不是特別為兒童設計，也不應用於傳送兒童個資或任何人的敏感資料。監護人及自架營運者應依適用法律評估年齡與同意要求。

## 8. 變更

重大政策變更會更新版本／生效日期並隨 source 發布。第三方修改版必須清楚提供其自己的隱私揭露，不得暗示沿用官方營運保證。

## 9. 法律提醒

本政策描述專案設計，不是法律意見，也不保證符合每個司法管轄區。正式公開或營運前，應由合格律師依實際網域、託管商、日誌、保留時間、使用者所在地及商店申報內容審閱。
